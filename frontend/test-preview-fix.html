급여 Preview 버튼 활성화 테스트

테스트 방법:

  1. 브라우저에서 http://localhost:3727 열기
  2. Admin (admin/admin)으로 로그인
  3. 급여 관리 → Excel 업로드 페이지로 이동
  4. 6월 데이터 파일 업로드 (모든 레코드가 duplicate/unmatched)
  5. Console 로그 확인 (F12 → Console)

예상 콘솔 출력:

📊 Preview Records Auto-Selection: {
  total: 9,
  selected: 9,    // ← 이제 9개 모두 선택됨!
  invalid: 0,
  unmatched: 3,
  statuses: {
    duplicate: 6,
    unmatched: 3
  }
}

✅ 수정 내용:

  • 기존: valid/warning 상태만 자동 선택
  • 수정: invalid를 제외한 모든 레코드 자동 선택
  • duplicate 레코드도 이제 선택됨 (업데이트 가능)
  • unmatched 레코드도 선택됨 (수동 매칭 가능)

테스트 시나리오:

파일 상태 분포 기존 동작 수정 후 동작
7월 데이터 valid: 7, unmatched: 1 7개 선택 → 버튼 활성화 ✅ 8개 선택 → 버튼 활성화 ✅
6월 데이터 duplicate: 6, unmatched: 3 0개 선택 → 버튼 비활성화 ❌ 9개 선택 → 버튼 활성화 ✅
